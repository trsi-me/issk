# درع الإنترنت للأطفال (ISSK)

تطبيق توعوي تعليمي يعمل بالكامل دون اتصال بالإنترنت، يهدف إلى تعريف الأطفال بمخاطر الإنترنت بطريقة تفاعلية، مع إمكانية لولي الأمر متابعة التقدم محلياً عبر قاعدة بيانات SQLite.

## الفكرة

- مسار للطفل: فيديوهات تعليمية مصنّفة، ألعاب بأربعة مستويات تُفتح بالتسلسل، واختبارات متعددة مع حفظ النتائج.
- مسار لولي الأمر: إنشاء رمز دخول، ثم عرض ملخصات وتفاصيل تقدم كل طفل مسجّل.

## التقنيات

- **Flutter / Dart** واجهة المستخدم فقط.
- **sqflite** لتخزين الأطفال، الفيديوهات، الألعاب، الأسئلة، التقدم، ورمز ولي الأمر.
- **video_player** و **chewie** لتشغيل الفيديو من الأصول؛ عند غياب الملف تظهر رسالة ودية دون تعطيل التطبيق.
- **intl** لتنسيق التواريخ في شاشات التقارير.
- **Python 3.10.6** (اختياري) لأي سكربتات مساعدة خارج التطبيق؛ التشغيل الأساسي لا يعتمد على بايثون.

لا يُستخدم أي ذكاء اصطناعي أو واجهات برمجة خارجية أو خدمات سحابية.

## الخوارزميات (باختصار)

- **فتح مستويات الألعاب:** المستوى 1 متاح دائماً؛ للمستوى `n` يُتحقق من وجود سجل تقدم مكتمل للمستوى `n-1` لنفس الطفل.
- **الاختبارات:** لكل اختبار من 1 إلى 4 يُختار مستوى أسئلة مطابق، ويُسحب 5 أسئلة عشوائياً من الأسئلة المتاحة لهذا المستوى.
- **النجوم:** تُحسب من نسبة الإجابات الصحيحة أو النقاط حسب نوع النشاط، وتُخزّن كقيمة رقمية في جدول التقدم.

## قاعدة البيانات

الجداول: `children`, `videos`, `games`, `quizzes`, `progress`, `parent_pin`. يتم إنشاء الملف تلقائياً عند أول تشغيل مع بيانات أولية (ألعاب، فيديوهات حسب التصنيفات، وعشرات الأسئلة التوعوية).

## متطلبات التشغيل

- [Flutter SDK](https://docs.flutter.dev/get-started/install) (قناة مستقرة حديثة تدعم Dart 3).
- [Dart](https://dart.dev/get-dart) مضمن مع Flutter.
- محرر اختياري: Android Studio أو Visual Studio Code مع إضافات Flutter/Dart.

4. لتشغيل التطبيق على جهاز أو محاكي:

## إصدارات البيئة في هذا المشروع (مرجع)

هذه القيم مأخوذة من إعدادات المشروع الحالية؛ قد يحدّثها Flutter تلقائياً عند ترقية SDK.

| المكوّن | المرجع في المشروع |
|--------|-------------------|
| **Dart** | `environment` في `pubspec.yaml`: `>=3.0.0 <4.0.0` |
| **Gradle (المدمج عبر Wrapper)** | `android/gradle/wrapper/gradle-wrapper.properties` — حالياً **Gradle 8.12** (ملف التوزيعة `gradle-*-all.zip`) |
| **Android Gradle Plugin** | `android/settings.gradle.kts` — حالياً **8.7.3** |
| **Kotlin** | `android/settings.gradle.kts` — حالياً **2.1.0** |
| **Java (لأندرويد)** | `android/app/build.gradle.kts` — **Java 11** للتوافق مع التجميع |

> **ملاحظة عن «Gradle 8.5»:** أحياناً يُذكر رقم مثل 8.5 كمثال لعائلة Gradle 8.x. في هذا المشروع **الرقم الفعلي محكوم بملف الـ Wrapper** (مثلاً 8.12). لا تحتاج عادةً لتثبيت Gradle يدوياً من الموقع؛ يكفي تشغيل `./gradlew` أو `flutter build` فيسحب Gradle المناسب. إذا احتجت توافقاً مع إصدار محدد، عدّل `distributionUrl` بحذر واختبر `flutter build apk`.

## بنية المشروع وشرح الأكواد (أين ماذا؟)

```
issk/
├── lib/                          # كل كود Dart للتطبيق
│   ├── main.dart                 # نقطة الدخول: الثيم، الاتجاه RTL، الشاشة الأولى (Splash)
│   ├── database/
│   │   └── database_helper.dart  # إنشاء SQLite، الجداول، الاستعلامات، الهجرة، البذور الأولية
│   ├── models/                   # نماذج بيانات (من JSON/صف إلى كائن Dart)
│   ├── services/
│   │   └── app_session.dart      # جلسة بسيطة (مثلاً معرّف ولي الأمر)
│   ├── utils/                    # ألوان، خطوط، ثوابت، نصف أقطار واجهة
│   ├── widgets/                  # مكوّنات قابلة لإعادة الاستخدام (أزرار، شريط سفلي، إلخ)
│   └── screens/                  # الشاشات حسب المسار
│       ├── splash_screen.dart
│       ├── home_screen.dart      # اختيار: طفل أو ولي أمر
│       ├── child/                # تسجيل دخول الطفل، الصدفة، الألعاب، الاختبارات، الفيديو، الإعدادات
│       ├── parent/               # ولي الأمر: تسجيل، لوحة، أبناء، تقدم
│       ├── about/                # عن التطبيق
│       └── settings/             # إعدادات ولي الأمر، الخصوصية
├── android/, ios/, ...           # مشاريع المنصّات (أندرويد يستخدم Gradle كما فوق)
├── assets/                       # صور، فيديوهات اختيارية، خطوط IBM Plex Sans Arabic
├── pubspec.yaml                  # الاعتماديات (packages) والأصول والخطوط
└── test/                         # اختبارات وحدات/واجهة
```

### تدفّق التشغيل باختصار

1. **`main.dart`** يضبط `MaterialApp`، اللغة العربية، و`Directionality` باتجاه RTL.
2. **`SplashScreen`** تنتقل بعد مهلة إلى **`HomeScreen`**.
3. **طفل:** `ChildLoginScreen` → التحقق من PIN عبر `DatabaseHelper` → `ChildShell` (تبويبات: رئيسية، ألعاب، اختبارات، إعدادات).
4. **ولي أمر:** `ParentLoginScreen` → `ParentShell` (تبويبات: لوحة، أبناء، تقدم، إعدادات، … حسب التصميم).
5. **الألعاب:** `GamesScreen` يقرأ المستويات من قاعدة البيانات ويفتح `GamePlayScreen` حسب `gameType` (صح/خطأ، ترتيب، اختيار من متعدد، إلخ).
6. **الاختبارات:** `QuizMenuScreen` → `QuizScreen` يجلب أسئلة حسب المستوى ويحفظ النتيجة في `progress`.
7. **الفيديو:** قائمة وبحث من `videos`؛ التشغيل قد يمر عبر معرّف يوتيوب أو ملف محلي حسب البيانات المخزّنة.

## كيف تم «بناء» التطبيق (فكرة هندسية)

- **طبقة واجهة:** Flutter Widgets فقط؛ لا خادم ولا API خارجي للمنطق الأساسي.
- **طبقة بيانات:** `sqflite` + `DatabaseHelper` كمصدر حقيقة واحد؛ النماذج في `lib/models/` تعكس أعمدة الجداول.
- **جلسة ولي الأمر:** `shared_preferences` أو متغيّرات الجلسة البسيطة حسب `app_session.dart` (راجع الملف للتفاصيل).
- **الوسائط:** فيديو محلي عبر `video_player`/`chewie`؛ فيديو يوتيوب عبر `youtube_player_flutter` عند الحاجة (مع صلاحية الإنترنت في `AndroidManifest` إن وُجدت).
- **الجودة:** أوامر مثل `flutter analyze` و`flutter test` للتحقق قبل الإصدار.

## تثبيت الأدوات على ويندوز (بالتفصيل)

### 1) تثبيت Flutter SDK

1. من [تثبيت Flutter](https://docs.flutter.dev/get-started/install/windows) نزّل أحدث إصدار **مستقر (stable)**.
2. فك الضغط إلى مجلد ثابت، مثلاً `C:\src\flutter` (تجنّب مسارات بمسافات طويلة أو أحرف خاصة إن أمكن).
3. أضف `C:\src\flutter\bin` إلى **PATH** في متغيرات البيئة للمستخدم أو النظام.
4. افتح **PowerShell** أو **CMD** جديدة ونفّذ:
   ```bash
   flutter doctor
   ```
5. اتبع توصيات `flutter doctor` (موافقة ترخيص أندرويد، تثبيت Visual Studio لبناء ويندوز إن احتجت، إلخ).

### 2) Dart

Dart يأتي **مدمجاً مع Flutter**؛ لا حاجة لتثبيت منفصل عادةً. تحقق بالأمر:
```bash
dart --version
```

### 3) Android Studio (موصى به لأندرويد)

1. نزّل [Android Studio](https://developer.android.com/studio) وثبّته.
2. من **More Actions → SDK Manager**:
   - ثبّت **Android SDK Platform** حديثاً (مثلاً المستوى الذي يطلبه `compileSdk` عندك).
   - من تبويب **SDK Tools** فعّل: **Android SDK Build-Tools**, **Android SDK Command-line Tools**, **Android Emulator**, **Google USB Driver** (على ويندوز للهاتف الحقيقي).
3. من **Device Manager** أنشئ **Virtual Device** (محاكي) بصورة نظام حديثة (مثلاً Pixel + API 34) إن أردت الاختبار بدون هاتف.

### 4) Visual Studio Code

1. نزّل [VS Code](https://code.visualstudio.com/).
2. من Extensions ثبّت:
   - **Flutter** (يشمل دعم Dart).
3. اضبط مسار Flutter إن طُلب منك: **File → Preferences → Settings** وابحث عن `flutter.sdk`.

### 5) Gradle (8.x) في سياق هذا المشروع

- **لا تثبّت Gradle يدوياً من الموقع** إلا لأسباب خاصة؛ المشروع يستخدم **Gradle Wrapper** داخل `android/`.
- عند أول بناء أندرويد، يُحمَّل Gradle تلقائياً حسب `gradle-wrapper.properties`.
- إذا رأيت خطأ توافق بين **AGP** و**Gradle**، راجع [جدول التوافق](https://developer.android.com/build/releases/gradle-plugin) أو حدّث Flutter ثم أعد `flutter pub get` و`flutter clean`.

## تشغيل المشروع على محاكي (Android Emulator)

1. شغّل Android Studio → **Device Manager** → شغّل المحاكي الذي أنشأته (انتظر حتى يكتمل الإقلاع).
2. في الطرفية داخل مجلد المشروع:
   ```bash
   cd D:\VSCode\Projects\flutterprojects\issk
   flutter pub get
   flutter devices
   ```
   يجب أن يظهر المحاكي في القائمة.
3. شغّل:
   ```bash
   flutter run
   ```
4. اختر الجهاز إن سُئلت. للتطوير: اضغط `r` في الطرفية لـ **Hot Reload**، أو `R` لـ **Hot Restart**.

## تشغيل المشروع على هاتف حقيقي (أندرويد)

1. على الهاتف: **الإعدادات → خيارات المطوّر** فعّل **تصحيح أخطاء USB** (وأحياناً **تثبيت عبر USB**).
2. وصّل الهاتف بكابل USB جيد؛ على ويندوز قد تحتاج تعريف **Google USB Driver** من SDK Manager.
3. وافق على رسالة «السماح بتصحيح الأخطاء USB» على الشاشة.
4. نفّذ:
   ```bash
   flutter devices
   ```
   يجب أن يظهر الجهاز.
5. شغّل:
   ```bash
   flutter run
   ```

إذا لم يظهر الجهاز: جرّب منفذ USB آخر، أو ثبّت تعريف الهاتف من الشركة المصنّعة، أو نفّذ `adb devices` بعد تثبيت **Platform Tools** ضمن Android SDK.

## خلاصة سريعة للمبتدئ

1. ثبّت **Flutter** وشغّل `flutter doctor` حتى تصبح المشكلات الحرجة صفر أو معروفة.
2. ثبّت **Android Studio** + **SDK** + (اختياري) **محاكي**.
3. افتح المشروع في **VS Code** أو Android Studio، ثم `flutter pub get` ثم `flutter run`.
4. راجع **بنية المجلدات** أعلاه عند قراءة الكود؛ ابدأ من `main.dart` ثم `database_helper.dart` ثم الشاشة التي تهتم بها.

> **تذكير:** المحتوى من السطر الأول حتى «عدم توفر ملف فيديو» هو الوصف الأصلي للمشروع؛ الأقسام أعلاه توسّعه دون إزالة أي فقرة سابقة.
